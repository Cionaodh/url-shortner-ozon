# Запуск сервиса

> Для работы нужен установленный и запущенный Docker

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/Cionaodh/url-shortner-ozon.git
cd url-shortner-ozon
```

### 2. Создайте `.env` файл

```bash
cp .env.example .env
```

### 3. Запустите сервис

```bash
make url_go
```

или

```bash
docker compose up
```

---

# Конфигурация

Все настройки задаются в `.env` файле в корне проекта.

Пример есть в `.env.example`.

### Хранилище

Выбор хранилища задаётся параметром:

- `STORAGE_TYPE=memory` — хранение в памяти
- `STORAGE_TYPE=postgres` — хранение в PostgreSQL

# API

> Базовый URL: [http://localhost:8080](http://localhost:8080)

---

## POST `/` — создать короткую ссылку

### Запрос

```json
{
	"url": "https://example.com/some/very/long/url"
}
```

### Ответы

```json
// 200 OK
{
	"short_url": "abc123XYZ_"
}
```

```json
// 400 — ошибка запроса
{
	"error": "invalid request"
}
```

```json
// 400 — некорректный URL
{
	"error": "invalid url"
}
```

```json
// 500 — внутренняя ошибка
{
	"error": "internal server error"
}
```

### Пример

```bash
curl -X POST http://localhost:8080/ \
  -d '{"url": "https://google.com"}'
```

---

## GET `/{short}` — получить оригинальный URL

### Ответы

```json
// 200 OK
{
	"url": "https://google.com"
}
```

```json
// 400 — неверный формат ссылки
{
	"error": "invalid short link"
}
```

```json
// 404 — ссылка не найдена
{
	"error": "not found"
}
```

```json
// 500 — внутренняя ошибка
{
	"error": "internal server error"
}
```

### Пример

```bash
curl http://localhost:8080/abc123XYZ_
```

# Задание

Необходимо реализовать сервис, который должен предоставлять API по созданию сокращенных ссылок следующего формата:

Ссылка должна быть уникальной и на один оригинальный URL должна ссылаться только одна сокращенная ссылка.

- Ссылка должна быть длинной 10 символов
- Ссылка должна состоять из символов латинского алфавита в нижнем и верхнем регистре, цифр и символа \_ (подчеркивание)

Сервис должен быть написан на Go и принимать следующие запросы по http:

1. Метод Post, который будет сохранять оригинальный URL в базе и возвращать сокращённый
2. Метод Get, который будет принимать сокращённый URL и возвращать оригинальный URL

Решение должно быть предоставлено в «конечном виде», а именно:

- Сервис должен быть распространён в виде Docker-образа 
- В качестве хранилища ожидается использовать две реализации. Какое хранилище использовать, указывается параметром при запуске сервиса.  
  - Первое это postgresql.
  - Второе - самостоятельно написать пакет для хранения ссылок в памяти приложения.
- Покрыть реализованный функционал Unit-тестами
